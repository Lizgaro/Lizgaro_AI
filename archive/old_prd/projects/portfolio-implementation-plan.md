# Portfolio Implementation Plan — подробный пред-план сайта

Цель файла: дать Codex/Claude/другому AI чёткое ТЗ на создание русскоязычного портфолио Артёма Лизгаро без каши в голове. Главный PRD лежит в `projects/portfolio-prd.md`. Этот файл дополняет PRD как implementation-plan.

## 0. Главная задача

Создать MVP сайта-портфолио:

```txt
личность -> доверие -> навыки -> кейсы -> услуги -> CTA
```

Сайт должен ощущаться как:

```txt
тёмный founder/operator portfolio
+ AI/маркетинг
+ лаймовый акцент
+ крупная типографика
+ минимализм
+ движение без перегруза
```

## 1. Ключевой смысл сайта

Артём — не просто “делаю сайты”. Он соединяет:

```txt
AI
+ маркетинг
+ сайты
+ боты
+ воронки
+ SEO/GEO
+ соцсети
+ инфлюенсер-партнёрки
+ RevOps/GrowthOps
```

Главное обещание:

> Помогаю превратить идею, продукт или хаос в систему: сайт, бот, воронка, контент, трафик и первые заявки.

## 2. Целевая аудитория

### Аудитория 1 — фаундеры AI/SaaS

Боль:

- есть продукт, но нет трафика;
- непонятна упаковка;
- нет партнёрок;
- нет контента;
- нет воронки.

Что показать:

- Ambassador Hub;
- SEO/GEO;
- growth-мышление;
- сайт/бот/воронка.

### Аудитория 2 — эксперты и инфлюенсеры

Боль:

- есть аудитория, но нет продукта;
- нет понятной монетизации;
- нужен сайт/бот/AI-виджет;
- хочется стать фаундером продукта.

Что показать:

- Marineos / AI Beauty Try-On;
- партнёрские модели;
- сайты/лендинги;
- контент и соцсети.

### Аудитория 3 — малый бизнес / команды

Боль:

- маркетинг, продажи и разработка не связаны;
- есть хаос;
- нет системы;
- нет автоматизации.

Что показать:

- RevOps/Growth Operator;
- боты;
- воронка;
- growth sprint.

### Аудитория 4 — подписчики

Боль:

- много шума;
- хочется действия;
- интересны AI, спорт, саморазвитие, путь.

Что показать:

- философия;
- фазы;
- блог;
- Life OS.

## 3. Референс-ДНК

### Devsync

Взять:

- тёмный фон;
- огромный hero с именем;
- зелёный/лаймовый акцент;
- developer/operator вайб;
- skill-блок;
- сетку проектов;
- сервисы;
- процесс;
- мощный CTA в футере.

Не брать:

- чужого персонажа;
- точную структуру Bruno Simon;
- фейковые клиенты/отзывы.

### Portavia

Взять:

- крупную condensed-типографику;
- портрет/карточку в hero;
- навигацию-пилюлю;
- аккордеон услуг;
- большие image-card проекты;
- отзывы/FAQ/форма;
- лаймовый footer/акцент.

Не брать:

- дизайн как “digital designer” один-в-один;
- чужие картинки и цифры.

### Billie Duvalle

Взять:

- editorial feel;
- асимметричные карточки работ;
- огромные заголовки;
- много воздуха;
- минимальную навигацию;
- тёмную премиальность.

Не брать:

- fashion/agency vibe слишком буквально;
- хаотичную плавающую кнопку шаблона;
- чужие бренды/клиентов.

## 4. Рекомендуемый стек

Для MVP:

```txt
Next.js
TypeScript
Tailwind CSS
Framer Motion
```

Почему:

- удобно для лендинга и будущего блога;
- можно быстро собрать компоненты;
- хорошая SEO-база;
- легко расширять проект;
- Codex хорошо справляется с такой структурой.

Альтернатива, если нужен совсем простой static MVP:

```txt
Vite + React + TypeScript + Tailwind CSS + Framer Motion
```

## 8. Секции подробно

### Header

Компонент: `Header.tsx`

Состав:

- слева маленький avatar/монограмма `AL`;
- center/right nav;
- CTA button.

Навигация:

```txt
Обо мне
Что делаю
Кейсы
Услуги
Контакты
```

Поведение:

- sticky top;
- background blur;
- mobile menu.

### Hero

Компонент: `Hero.tsx`

Визуально:

- huge title `АРТЁМ ЛИЗГАРО`;
- тёмный фон;
- лаймовый glow;
- hero-card вместо чужого фото: можно использовать абстрактную карточку “AI / Growth / Systems”;
- справа/снизу короткое описание;
- tags.

Hero должен быть мощным, но не копировать Devsync/Portavia.

### About

Компонент: `About.tsx`

Смысл:

- путь из практической среды;
- переход в AI/маркетинг;
- способность видеть суть и собирать системы.

Не делать длинную автобиографию. 2–3 абзаца.

### ServicesAccordion

Компонент: `ServicesAccordion.tsx`

Взять идею аккордеона из Portavia:

- numbered list;
- раскрытие по клику;
- каждый сервис с 3–5 bullets;
- CTA внутри первого/активного сервиса.

### SkillsGrid

Компонент: `SkillsGrid.tsx`

Вдохновение Devsync:

- категории слева;
- tags справа;
- можно добавить короткий текст “не стек ради стека, а система”.

### CaseStudies

Компонент: `CaseStudies.tsx`

Вдохновение:

- Devsync: сетка проектов;
- Portavia: большие image cards;
- Billie: асимметричная editorial-сетка.

Решение v1:

- 5 карточек;
- 2 колонки на desktop;
- первая карточка шире;
- вместо реальных скриншотов использовать абстрактные gradient cards или placeholder graphics.

Каждая карточка:

```txt
label
title
short description
tags
status
CTA: Подробнее / Обсудить похожее
```

### Offers

Компонент: `Offers.tsx`

Карточки услуг:

- Консультация.
- Сайт/лендинг.
- Бот/парсер.
- Growth Sprint.
- AI MVP.

У каждой:

- кому подходит;
- что на выходе;
- CTA.

### Philosophy

Компонент: `Philosophy.tsx`

Короткие принципы:

- 5 минут мысли -> 50 минут действия;
- MVP раньше идеала;
- сначала аудитория;
- сначала руками, потом автоматизация;
- не потреблять — создавать.

### FAQ

Компонент: `FAQ.tsx`

Аккордеон вопросов.

## 10. Проверка качества

Перед завершением Codex должен проверить:

- сайт запускается;
- нет TypeScript ошибок;
- нет явных ошибок в консоли;
- адаптив на mobile/tablet/desktop;
- CTA видны;
- русский текст не ломает layout;
- нет фейковых отзывов/логотипов;
- референсы не скопированы один-в-один.

## 11. Запуск MVP

Минимальный порядок:

1. Создать проект.
2. Настроить Tailwind.
3. Создать data-файлы.
4. Собрать все секции без анимаций.
5. Добавить адаптив.
6. Добавить базовые анимации.
7. Проверить на мобильном.
8. Сделать README с запуском.
9. Сделать скриншоты результата.
10. Потом уже улучшать.

## 12. Что отложить

- Блог/CMS.
- Страницы отдельных кейсов.
- Интеграция формы.
- Сложный backend.
- CRM.
- 3D.
- Автоматический парсинг контента.
- Сложные custom cursors.
- Многостраничность.

## 13. Definition of Done

Готово, когда:

- сайт на русском;
- структура соответствует брифу;
- дизайн тёмный, минималистичный, лаймовый акцент;
- есть hero, about, services, skills, cases, offers, philosophy, FAQ, contact;
- работает адаптив;
- есть инструкция запуска;
- нет фейковых метрик;
- сайт можно показать клиенту.
