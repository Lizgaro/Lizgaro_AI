## Goal

Создать современный portfolio landing page на русском языке для Артёма Лизгаро.

Сайт должен продавать:

- личные консультации;
- создание сайтов/лендингов;
- Telegram-ботов и парсер-ботов;
- маркетинговые стратегии и воронки;
- SEO/GEO и органический рост;
- масштабирование бизнеса через Growth/RevOps;
- AI/MVP и партнёрские связки.

## Context

Артём — AI-маркетолог, builder и growth-оператор. Он соединяет AI, маркетинг, сайты, боты, воронки, контент, соцсети, SEO/GEO и партнёрства.

Ключевое позиционирование:

```txt
Артём Лизгаро
AI-маркетолог, builder и growth-оператор.
Создаю сайты, боты, воронки и AI-системы, которые помогают бизнесам и проектам расти.
```

Ключевая фраза:

```txt
От хаоса идей к сайтам, ботам, трафику и продажам.
```

## Design Direction

Сделай стиль:

```txt
Dark AI Operator + Neon Lime + Editorial Portfolio
```

Используй референсы как дизайн-ДНК, но не копируй:

- Devsync: dark developer vibe, big hero, neon accent, skill table, project grid, services, final CTA.
- Portavia: bold condensed typography, pill nav, services accordion, big project cards, FAQ/contact form, lime accent.
- Billie Duvalle: editorial asymmetry, huge headings, premium dark minimalism, selected works layout.

Запрещено:

- копировать композицию один-в-один;
- копировать тексты;
- использовать чужие изображения;
- выдумывать отзывы/клиентов/метрики;
- делать шаблонный “дизайнерский” сайт без сути Артёма.

## Recommended Stack

Используй:

```txt
Next.js + TypeScript + Tailwind CSS + Framer Motion
```

Если проект уже создан на другом стеке, адаптируй под него, но сохрани структуру и стиль.

## Visual Requirements

- Тёмный фон.
- Лаймовый/зелёный акцент.
- Большие заголовки.
- Сильный hero.
- Карточки с мягкими границами.
- Responsive mobile-first.
- Никакого visual noise.
- Мягкие animations, не перегружать.
- Стиль должен быть современным, но не “слишком шаблонным”.

## Implementation Requirements

1. Вынеси контент в `data/`.
2. Сделай компоненты по секциям.
3. Добавь базовые SEO metadata.
4. Сделай адаптив.
5. Добавь README с командами запуска.
6. Добавь проверку `npm run lint` / `npm run build`, если доступно.
7. Не добавляй новые тяжёлые зависимости без причины.

## Done When

- Есть рабочий сайт.
- Он запускается локально.
- Все основные секции есть.
- Текст на русском.
- CTA видны.
- Нет фейковых отзывов/клиентов.
- Дизайн вдохновлён референсами, но не копирует их.
- Есть README с запуском.
- Код разбит на понятные компоненты.

## First Step

Сначала не кодь. Составь короткий план реализации:

```txt
1. Какие файлы создашь
2. Какие секции соберёшь
3. Как организуешь data
4. Какие проверки запустишь
```

После плана приступай к реализации.
